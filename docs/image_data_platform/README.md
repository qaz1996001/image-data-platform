# image_data_platform 專案文檔系統 (Quick Start)

> **image_data_platform Project Documentation System**  
> 基於 `meta-framework` 的標準化醫療器材軟體開發文檔系統。

---

## 🎯 快速導覽

1. **查看索引**：[`00_IMAGE_DATA_PLATFORM_INDEX.md`](00_IMAGE_DATA_PLATFORM_INDEX.md)（了解完整結構）
2. **查看模板**：[`templates/00_TEMPLATES_INDEX.md`](templates/00_TEMPLATES_INDEX.md)（為各類文件選擇標準模板）
3. **需求管理**：[`requirements/00_REQUIREMENTS_INDEX.md`](requirements/00_REQUIREMENTS_INDEX.md)
4. **架構設計**：[`architecture/00_ARCHITECTURE_INDEX.md`](architecture/00_ARCHITECTURE_INDEX.md)
5. **合規證明**：[`regulations/00_REGULATIONS_INDEX.md`](regulations/00_REGULATIONS_INDEX.md)
6. **追溯矩陣**：[`traceability/00_TRACEABILITY_INDEX.md`](traceability/00_TRACEABILITY_INDEX.md)

---

## 🚀 核心開發流程

1. **需求變更**：更新 `requirements/` 下的文檔。
2. **OpenSpec 提案**：執行 `/openspec-proposal` 建立變更。
3. **開發實作**：遵循 `tasks.md` 並更新對應的 `architecture/` 文檔。
4. **驗證歸檔**：通過測試後執行 `openspec archive`。
