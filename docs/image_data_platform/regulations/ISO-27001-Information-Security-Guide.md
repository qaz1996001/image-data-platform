## 1. 資訊安全管理體系 (ISMS) 範圍

`image_data_platform` 的資訊安全範圍涵蓋儲存、處理患者腦部 MRI 影像的軟體系統及其相關開發環境。

## 2. 安全控制措施

### 2.1 存取控制 (Access Control)
- 軟體系統必須具備身份驗證機制。
- 遵循最小權限原則 (Least Privilege)。

### 2.2 密碼學控制
- 敏感資料 (如患者個資) 在傳輸與儲存時必須進行加密 (AES-256)。

### 2.3 操作安全
- 定期進行弱點掃描。
- 建立安全開發週期 (Security Development Lifecycle)。

### 2.4 資料隱私
- MRI 影像在進入 AI 訓練或推論系統前，必須通過去識別化程序，移除 DICOM 標籤中的姓名、身分證字號等資訊。

## 3. 事故處理
- 建立資安事故報告流程，記錄並回應任何未經授權的存取或資料外洩。
