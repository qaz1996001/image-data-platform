# image_data_platform 需求關鍵字使用指南 (RFC 2119)

**文件 ID**: DOC-IMAGE-DATA-PLATFORM-REG-017  
**標題**: image_data_platform 專案需求描述語法規範  
**版本**: v1.0.0

## 1. 目的

為確保 image_data_platform 專案在醫療器材軟體開發過程中的需求定義清晰、無歧義，本專案採用 **RFC 2119** 關鍵字作為需求描述的標準語法。

## 2. 專案特定範例

### 2.1 MUST (絕對要求)
*   系統 **MUST** 符合 IEC 62304 軟體安全等級分類。
*   影像處理模組 **MUST** 支援 DICOM 標準格式。

### 2.2 SHOULD (強烈建議)
*   演算法執行進度 **SHOULD** 在 1 秒內更新一次 UI。
*   日誌記錄 **SHOULD** 包含操作員 ID 與時間戳記。

### 2.3 MAY (可選功能)
*   使用者 **MAY** 自定義報告的浮水印樣式。
*   系統 **MAY** 支援導出為 PDF 以外的格式。

## 3. 實施要求

1. **所有人與 AI**：在撰寫新的需求或修改現有需求時，必須嚴格遵守此語法。
2. **OpenSpec 變更**：所有變更提案中的 spec deltas 必須通過 RFC 2119 語法檢查。
