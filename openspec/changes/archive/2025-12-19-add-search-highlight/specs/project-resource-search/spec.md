## ADDED Requirements

### Requirement: 專案資源搜尋結果高亮顯示
系統 SHALL 在專案資源列表與專案資源詳情頁面中高亮顯示符合搜尋條件的文字，以黃色背景和加粗字體標記，幫助使用者快速定位相關資訊。

#### Scenario: 基本搜尋文字高亮
- **GIVEN** 使用者在專案資源頁面輸入基本搜尋關鍵字「胸部」
- **WHEN** 使用者點擊某個資源查看詳情
- **THEN** 系統在資源詳情頁面中將所有包含「胸部」的文字以黃色背景 (#fff566) 和加粗字體顯示

#### Scenario: 搜尋結果列表欄位高亮
- **GIVEN** 使用者在專案資源頁面輸入搜尋關鍵字「31410270375」
- **WHEN** 系統顯示資源列表
- **THEN** 系統在列表可見欄位（例如 Accession No、Study Info、Report Info、Snippet）中將符合文字以黃色背景 (#fff566) 和加粗字體顯示

#### Scenario: 進階搜尋多欄位高亮
- **GIVEN** 使用者使用進階搜尋輸入檢查編號「S456」和病患ID「P789」
- **WHEN** 使用者點擊某個資源查看詳情
- **THEN** 系統在資源詳情頁面中同時高亮顯示「S456」和「P789」
- **AND** 不同關鍵字使用相同的視覺樣式

#### Scenario: 多條件進階搜尋高亮
- **GIVEN** 使用者使用多條件進階搜尋設定多個篩選條件
- **WHEN** 使用者點擊某個資源查看詳情
- **THEN** 系統高亮顯示所有符合搜尋條件的文字內容
- **AND** 高亮樣式與其他搜尋模組保持一致

#### Scenario: 大小寫不敏感匹配
- **GIVEN** 使用者搜尋關鍵字「CT」
- **WHEN** 資源內容包含「ct」、「CT」或「Ct」
- **THEN** 系統高亮顯示所有大小寫變體的匹配文字

#### Scenario: 無搜尋條件時不高亮
- **GIVEN** 使用者直接從專案頁面進入資源詳情（無搜尋條件）
- **WHEN** 查看資源詳情
- **THEN** 系統不進行任何文字高亮，正常顯示內容

#### Scenario: Snippet 高亮樣式一致
- **GIVEN** 系統在資源列表/詳情顯示 Snippet（可能包含 HTML 標記）
- **WHEN** Snippet 內容包含符合搜尋條件的文字
- **THEN** 系統以同樣的高亮樣式（黃色背景 #fff566 + 加粗字體）顯示匹配文字（而非依賴不同的 HTML 高亮樣式）
